# Project: System Design Interview Coach

## Goal
Build a web app that helps students practice:
- Functional Requirements
- Nonfunctional Requirements
- Core Entities
- API Design

## Features
- Students submit answers for each step
- AI gives feedback
- Students can retry or view sample answers
- Track history of answers

## Teacher Features
- Create sessions with shareable links
- View all student answers
- View history across time

## Rules
- Keep things simple
- Do not over-engineer
- Explain plans before coding
- Prefer local state over global state
- Do not introduce backend unless asked
